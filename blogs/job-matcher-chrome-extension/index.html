Building an AI-Powered LinkedIn & Naukri Job Matcher — Chrome Extension

June 2025 • 10 min read • Chrome Extension, DeepSeek V3, Google Sheets

Job hunting is tedious. You scroll through dozens of listings, copy-paste descriptions, and manually compare them against your CV. I built a Chrome extension that automates the entire workflow — extracting job descriptions, running AI-powered CV matching via DeepSeek V3, and saving results to Google Sheets. All client-side, zero ban risk, for ~$1.50/month.

The Problem

LinkedIn and Naukri show hundreds of jobs, but there's no built-in way to check how well a role matches your specific skills. You either maintain a manual spreadsheet (time-consuming) or rely on gut feel (error-prone). I wanted something that would:

  • Auto-detect when I'm on a job posting page
  • Extract the job description without copy-pasting
  • Compare it intelligently against my CV using an LLM
  • Save matched jobs to a spreadsheet I can review later

Architecture

The extension uses Chrome Manifest V3 with three components:

ComponentRole
Content ScriptInjected into LinkedIn/Naukri job pages. Detects URL patterns, extracts JD via site-specific DOM selectors, renders a floating blue button + analysis panel.
Background Service WorkerHandles API calls to DeepSeek for AI analysis and Google Apps Script Web App for saving to Sheets. Keeps API keys in chrome.storage.
Popup UISettings panel for configuring the DeepSeek API key and Google Sheets Web App URL, with a test button for validation.

Data Flow

Job Page → Content Script (DOM extraction) → Background Worker
                                                      ├── DeepSeek API (CV vs JD analysis)
                                                      └── Google Apps Script (append row to Sheet)

AI Response (structured JSON):
  matchPercentage: 75%
  matchedSkills:   [Java, Spring Boot, AWS, Redis, ...]
  missingSkills:   [Kubernetes, Terraform]
  strengths:       ["6+ years Java/Spring Boot", ...]
  gaps:            ["No K8s production experience"]
  summary:         "Strong backend match, minor cloud orchestration gaps"

Job Extraction

Site-specific DOM selectors extract these fields from LinkedIn and Naukri job pages:

  • Title — from h1 / title elements
  • Company — from company name links
  • Location — from location spans/anchors
  • Posted date — from date elements
  • Description — from the job description block, cleaned of "About the job" prefixes and collapsed newlines
  • URL — window.location.href

SPA-Aware Injection

LinkedIn is a Single Page Application — navigating between jobs doesn't trigger page reloads. This posed a challenge: the extension needs to detect when the user moves to a new job and re-inject the floating button + analysis panel.

Solution: the content script polls window.location.href every second. On URL change, it removes the old overlay and injects a fresh one, resetting all state (analysisResult, isAnalyzing, isSaving). A MutationObserver also watches for LinkedIn removing the overlay during DOM updates and re-injects it.

AI Analysis — Why DeepSeek V3?

I chose DeepSeek V3 over GPT-4 for one reason: cost. Here's the breakdown:

MetricDeepSeek V3GPT-4
Input cost (per 1M tokens)$0.27$30.00
Output cost (per 1M tokens)$1.10$60.00
Per-analysis cost~$0.001~$0.06
100 jobs/day / month~$1.50~$90.00

For structured JSON output (match percentage, skill lists, summary), DeepSeek V3 provides comparable quality at roughly 1/60th the cost. The API call sends ~2,400 tokens per analysis (1,600 for JD + 600 for CV + 200 prompt overhead) with temperature: 0.2 for consistent results.

Design Decision: The prompt instructs DeepSeek to return strictly structured JSON. This makes the response parseable and avoids LLM hallucination on format. The extension validates the JSON before rendering.

Google Sheets Integration

Instead of building a backend with OAuth, I used a Google Apps Script Web App deployed with "Anyone" access. The extension POSTs job data as text/plain to avoid CORS preflight (which would add latency and complexity).

The Apps Script (apps-script.gs) parses the JSON payload and appends a row with 11 columns:

ColumnDescription
TitleJob title
CompanyCompany name
LocationJob location
URLDirect link to job posting
PostedPosted date
DescriptionFirst 5,000 chars of cleaned JD
Match %AI-computed match percentage
Matched SkillsCV skills found in JD
Missing SkillsJD skills not in CV
SummaryOne-line AI assessment
Saved AtTimestamp

This gives a searchable, sortable job tracker that lives in Google Drive — no database, no server, no cost.

Troubleshooting Common Issues

From real usage, here are the most common issues and fixes:

IssueFix
Blue button doesn't appearEnsure you're on a LinkedIn /jobs/view/* page or Naukri job detail page. Refresh if the page hasn't fully loaded.
"Could not extract job description"Scroll down to trigger lazy-load of the description. New LinkedIn layouts may require selector updates.
Save to Sheet failsVerify the Apps Script Web App URL in extension settings. Re-deploy with "Anyone" access if needed.
Panel doesn't reset between jobsThe panel polls URL every 1s — wait a moment. If LinkedIn's DOM removes it, MutationObserver re-injects it.

Cost & Scale

Total monthly operating cost: ~$1.50 (DeepSeek API only). Google Sheets is free, the Chrome extension runs locally, and there's no server to maintain. At 200+ analyses per day, cost stays under $3/month.
